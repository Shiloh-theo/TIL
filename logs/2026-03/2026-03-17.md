## Topic: OCR in Java — Tesseract + Tess4J

### What is OCR?
Optical Character Recognition — converts images containing text into machine-readable text. Works in 5 stages: image preprocessing → layout analysis → text line/word detection → character recognition → post processing.

### The Java OCR Stack
- **Tess4J** — Java wrapper around Google's Tesseract OCR engine
- **Apache PDFBox** — converts PDF pages to BufferedImage before OCR processing
- **OpenCV** (advanced) — image preprocessing for low-quality inputs

### Key Concept: Page Segmentation Modes (PSM)
Tells Tesseract what layout to expect. Wrong PSM = garbage output.
- PSM 3 — fully automatic (default, good for full pages)
- PSM 6 — single uniform block of text
- PSM 11 — sparse text, best for receipts and mixed layouts

### Key Concept: Tessdata
Tesseract requires language data files (`eng.traineddata`) to recognise characters. Source: https://github.com/tesseract-ocr/tessdata

### PDF OCR Flow
PDF → PDFBox renders each page as BufferedImage at 300 DPI → save as temp PNG → pass to Tess4J → extract text per page → combine output

### Accuracy Insight
80% of OCR accuracy is determined at the preprocessing stage, not the recognition stage. A blurry image fed into the best engine still gives poor results.
